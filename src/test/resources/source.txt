            <h2 class="entry-title"><a href="https://blog.jetbrains.com/kotlin/2018/03/kotlin-1-2-30-is-out/" rel="bookmark">Kotlin 1.2.30 is out</a></h2>

            <div class="entry-meta">
                <span class="meta-prep meta-prep-author"> Posted on </span> <a href="https://blog.jetbrains.com/kotlin/2018/03/kotlin-1-2-30-is-out/" title="9:11 pm" rel="bookmark"><span class="entry-date">March 1, 2018</span></a> <span class="meta-sep">by</span> <span class="author vcard"><a class="url fn n" href="https://blog.jetbrains.com/kotlin/author/sergeyigushkin/" title="View all posts by Sergey Igushkin">Sergey Igushkin</a></span>            </div><!-- .entry-meta -->

                            <div class="entry-content">
                    <p>We’re happy to announce the release of Kotlin 1.2.30, a new bugfix and tooling update for Kotlin 1.2. This update:</p>
<ul>
<li>Adds a new declaration in the standard library, which imitates the <code>suspend</code> modifier for lambda expressions</li>
<li>Adds support for TestNG in <code>kotlin.test</code></li>
<li>Brings support for Android modules in multiplatform projects</li>
<li>Introduces a new feature in kapt for reporting annotation processing errors along with proper links to the original Kotlin declarations</li>
<li>Adds a lot of new inspections and intentions in the IntelliJ plugin and improves its performance</li>
<li>Fixes bugs in the compiler and IntelliJ plugin</li>
</ul>
<p>The update is compatible with all versions of IntelliJ IDEA from 2017.1 until 2017.3 and 2018.1 EAP, as well as with Android Studio 3.0 and Android Studio 3.1 (Beta)</p>

<li>CLion plugin update for CLion 2018.1 (fixed the issue with macOS platform libraries indexer)</li>
</ul>

            <h2 class="entry-title"><a href="https://blog.jetbrains.com/kotlin/2018/02/using-gradle-build-cache-with-kotlin/" rel="bookmark">Using Gradle build cache with Kotlin</a></h2>

            <div class="entry-meta">
                <span class="meta-prep meta-prep-author"> Posted on </span> <a href="https://blog.jetbrains.com/kotlin/2018/02/using-gradle-build-cache-with-kotlin/" title="6:29 pm" rel="bookmark"><span class="entry-date">February 6, 2018</span></a> <span class="meta-sep">by</span> <span class="author vcard"><a class="url fn n" href="https://blog.jetbrains.com/kotlin/author/roman-belov/" title="View all posts by Roman Belov">Roman Belov</a></span>            </div><!-- .entry-meta -->

                            <div class="entry-content">
                    <p><center></p>
<div style="background-color: #f1f6fe; margin-bottom: 40px; padding: 15px; line-height: 20px; text-align: justify; overflow: auto; width: 500px;">
<p><img style="float: left; margin-right: 15px; margin-top: 0px;" title="Eric Wendelin" src="https://avatars3.githubusercontent.com/u/51534" alt="Eric Wendelin" width="70" height="70" /><br />
This is a guest blog post from <a href="https://github.com/eriwen">Eric Wendelin</a><br />
<i>software engineer for Gradle</i>
</div>
<p></center></p>
<p>A <a href="https://blog.gradle.org/introducing-gradle-build-cache">build cache</a> allows Gradle to reuse task output from <em>any previous invocation, including those from other machines</em>. <a href="https://blog.jetbrains.com/kotlin/2018/01/kotlin-1-2-20-is-out/">Kotlin 1.2.21</a> allows Kotlin projects to make use of build caching.</p>
<p>The build cache works by storing compiled classes, test outputs, and other build artifacts in a cache, taking into account all task inputs, including input file contents, relevant classpaths, and task configuration.</p>
<p><img class="alignnone size-full wp-image-5722" src="https://d3nmt5vlzunoa1.cloudfront.net/kotlin/files/2018/02/build-cache-topological-diagram.png" alt="Build Cache topological diagram" /></p>
<p>This frequently results in <em>faster builds</em>. The following chart shows aggregated build time with and without the build cache for part of Gradle&#8217;s CI:</p>

<p>In this post, we&#8217;ll explain how you can use Gradle&#8217;s build cache to avoid unnecessary Kotlin compilation to speed up your builds.<br />

<li>Improves incremental compilation for Android and mixed Kotlin/Java projects</li>

            <h2 class="entry-title"><a href="https://blog.jetbrains.com/kotlin/2017/11/kotlin-1-2-released/" rel="bookmark">Kotlin 1.2 Released: Sharing Code between Platforms</a></h2>

            <div class="entry-meta">
                <span class="meta-prep meta-prep-author"> Posted on </span> <a href="https://blog.jetbrains.com/kotlin/2017/11/kotlin-1-2-released/" title="7:52 pm" rel="bookmark"><span class="entry-date">November 28, 2017</span></a> <span class="meta-sep">by</span> <span class="author vcard"><a class="url fn n" href="https://blog.jetbrains.com/kotlin/author/dmitry-jemerov/" title="View all posts by Dmitry Jemerov">Dmitry Jemerov</a></span>            </div><!-- .entry-meta -->

<div style="margin:-30px 0px 10px 0px;padding:10px;border-color:#f0f0f0;display:inline-block;border-width:1px;border-style:solid;font-size:10">Members of our community have translated<br />

<p>Today we&#8217;re releasing Kotlin 1.2. This is a major new release and a big step on our road towards enabling the use of Kotlin across all components of a modern application.</p>
<p>In Kotlin 1.1, we officially released the <strong>JavaScript target</strong>, allowing you to compile Kotlin code to JS and to run it in your browser. In Kotlin 1.2, we&#8217;re adding the possibility to <strong>reuse code between the JVM and JavaScript</strong>. Now you can write the business logic of your application once, and reuse it across all tiers of your application &#8211; the backend, the browser frontend and the Android mobile app. We&#8217;re also working on libraries to help you reuse more of the code, such as a cross-platform serialization library.</p>
